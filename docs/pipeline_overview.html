Final pipeline architecture

Video-to-Chat Surveillance QA

A pipeline-first system: the backend first turns raw video into structured, time-stamped memory, then user questions retrieve relevant evidence and keyframes before Gemma4 generates a grounded answer.

YOLO + BoT-SORT for people
SAM3 for objects
5s event windows
Gemma4 VLM memory
Interval summaries disabled
Core idea
Process first, then chat.

Gemma4 is not asked to understand the entire raw video alone. It answers from retrieved memory: tracks, object associations, window captions, stitched timelines, and sampled keyframes.

Ingest
Uploaded Video
Stored video file from the Streamlit frontend.
Backend Registry
Assigns video id, camera id, path, and metadata.
Chat Session + Job
Creates session id, job id, expected run id, and processing status.
Background Worker
Runs the full batch pipeline before chat unlocks.
Process
Frame Sampler
OpenCV samples frames at 4 FPS and preserves timestamps.
Person Tracking
YOLO + BoT-SORT detects people and assigns internal tracker IDs.
Appearance Memory
Reassociates fragmented person tracks into stable internal identities.
Object Backend
SAM3 detects and segments configured object classes.
Associations
Builds person-object proximity and person-person interaction evidence.
Memory
5s Event Windows
Stores active tracks, timestamps, events, objects, and interactions.
Visual Evidence
Collects track crops, window keyframes, and sampled scene frames.
Gemma4 Track Profiles
Describes appearance and behavior for each internal person track.
Gemma4 5s Captions
Captioned window-level activity from keyframes and structured events.
Full Scene Summary
Gemma4 summarizes the whole video from sampled frames.
Store
SQLite Run Memory: videos, jobs, tracks, events, summaries, profiles, chat sessions, and messages
Query
User Question
Chat unlocks only after backend processing is complete.
RAG Retrieval
Builds time-stamped documents from SQLite memory on demand.
Stitched Timeline
Connects adjacent windows for the same internal person identity.
Sample Keyframes
Reads visual evidence from the retrieved time windows.
Gemma4 Final Answer
Concise grounded answer without exposing internal track IDs.
Ablations Tested, then removed or demoted from the final path

60s intervalsToo coarse for person-level questions.
Qwen3-VLEarlier baseline, replaced by Gemma4 final.

Text-only QAFaster but misses visual detail.
Current system is pipeline-first, not real-time. Future work: commit completed 5-second windows incrementally for near-real-time QA.
